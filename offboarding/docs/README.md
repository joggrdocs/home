# Offboarding Joggr Docs

Disconnect your repositories from Joggr Docs: uninstall the GitHub App, then strip Joggr-managed HTML comments out of your markdown files. Your Joggr account, document history on app.joggr.io, and any internal data Joggr holds about your team are out of scope here — contact the team in Slack if you want those deleted.

## Two paths

- **Manual** — uninstall the GitHub App, then follow the [step-by-step instructions](#manual-steps) to remove markers by hand.
- **Automatic** — uninstall the GitHub App, then run the [Node scripts](#scripts). Per-file verbatim backups, one-shot, with a `restore.mjs` to undo it.

The automatic scripts cover the **happy path only**. If they hit anything unexpected — uncommitted git changes, an existing backup directory, a file with malformed markers, a hash mismatch — they fail and tell you to use the manual steps. There is no recovery logic, no resume, no override flags. Keep it simple, ship the rollback.

> [!WARNING]
> The scripts rewrite markdown files in place inside the scan root you pass them. Provided as-is, run at your own risk. Tested on macOS with Node 20+. Always run `node offboard.mjs --dry-run` first.

## Quick start

If you've already downloaded this directory (via `install.sh` or by cloning the repo) and are in it:

```bash
node offboard.mjs --dry-run     # preview what would change (no disk writes)
node offboard.mjs               # interactive strip with per-file backups
node restore.mjs                # undo
```

The scripts default to the **enclosing git repo** as the scan root — if you've dropped this directory inside your docs repo via `install.sh`, the right thing happens automatically. Pass `--scan <path>` to override.

`npm run` shortcuts (`npm run offboard`, `npm run restore`) exist but use the direct `node` form when passing flags — `npm` silently consumes top-level flags like `--dry-run` unless you use the `--` forwarding syntax.

## What Joggr Docs installs

| Where                      | What                                                                                                                     | Handled by scripts?              |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------ | -------------------------------- |
| GitHub                     | Joggr GitHub App install with repo permissions                                                                            | Manual (step 1)                  |
| Each managed markdown file | File marker: `<!--@@joggrdoc@@-->`                                                                                       | ✓ line removed                   |
| Each managed markdown file | Version stamp: `<!-- @joggr:version(vN):end -->`                                                                          | ✓ line removed                   |
| Each managed markdown file | Block: `<!-- @joggr:warning:start -->` ... `:end -->` (auto-generated "managed by Joggr" banner)                          | ✓ block + content removed        |
| Each managed markdown file | Block: `<!-- @joggr:editLink(<uuid>):start -->` ... `:end -->` (auto-generated "Edit on Joggr" badge)                      | ✓ block + content removed        |
| Each managed markdown file | Block: `<!-- @joggr:snippet(<id>):start -->` ... `:end -->` (wraps user code synced from source)                           | ✓ **markers only**, code kept    |
| Each managed markdown file | Block: `<!-- @joggr:createSnippet(<meta>):start -->` ... `:end -->`                                                        | ✓ **markers only**, code kept    |
| Each managed markdown file | Block: `<!-- @joggr:autoTarget(<id>):start -->` ... `:end -->`                                                             | ✓ **markers only**, content kept |
| Each managed markdown file | Block: `<!-- @joggr:codeLink(<id>):start -->` and `<!-- @joggr:createCodeLink(...):start -->` ... `:end -->`               | ✓ **markers only**, content kept |
| app.joggr.io               | Your documents, history, team data — these live on Joggr's servers                                                        | Out of scope                     |

**Critically:** the trimmer only deletes content for `warning` and `editLink` blocks — the two blocks whose content is auto-generated by Joggr and has no value after offboarding. Every other block type wraps **your code or content** and is unwrapped without touching what's inside. This matches the upstream behavior of `@joggrdocs/bashir`'s `stripAll` and is verified against bashir's fixture files.

The scripts only touch `.md` and `.mdx` files inside the scan root. They never call out to GitHub.

## Manual steps

### 1. Uninstall the Joggr GitHub App

Joggr Docs writes back to your repositories through a GitHub App install. Uninstalling it revokes Joggr's access; existing comments in your files are left alone until you remove them in step 2.

**For a personal account:**

1. Open <https://github.com/settings/installations>
2. Find **Joggr** and click **Configure**.
3. Scroll to the **Danger zone** and click **Uninstall**.

**For an organization:**

1. Open `https://github.com/organizations/<your-org>/settings/installations`
2. Find **Joggr** and click **Configure**.
3. Scroll to the **Danger zone** and click **Uninstall**.

GitHub's docs walk through the same flow with screenshots: <https://docs.github.com/en/apps/using-github-apps/reviewing-and-modifying-installed-github-apps#uninstalling-a-github-app>.

You may also want to revoke the OAuth authorization at <https://github.com/settings/applications>: find **Joggr** under **Authorized OAuth Apps** and revoke it.

> [!IMPORTANT]
> Uninstalling the GitHub App stops Joggr from being able to push commits to your repos. Existing Joggr-authored commits and the HTML comments already in your files are not undone by this step — that's what step 2 is for.

### 2. Identify Joggr-managed files

```bash
# Precise: only Joggr-managed files (matches the file marker).
grep -rl '<!--@@joggrdoc@@-->' . --include='*.md' --include='*.mdx'

# Broader: any markdown that contains the substring `@joggr:`.
grep -rl '@joggr:' . --include='*.md' --include='*.mdx'
```

### 3. Remove the markers from each file

For each managed file, four edits in order. **Back up first:**

```bash
cp path/to/doc.md path/to/doc.md.bak
${EDITOR:-vi} path/to/doc.md
```

Then:

1. **Delete the file marker** on line 1: `<!--@@joggrdoc@@-->`.
2. **Delete the version stamp** line: `<!-- @joggr:version(vN):end -->`.
3. **Delete the warning block** — everything from `<!-- @joggr:warning:start -->` through `<!-- @joggr:warning:end -->` inclusive.
4. **Delete the editLink block** — everything from `<!-- @joggr:editLink(<uuid>):start -->` through `<!-- @joggr:editLink(<uuid>):end -->` inclusive.

For other block types (`snippet`, `createSnippet`, `autoTarget`, `codeLink`, `createCodeLink`): delete the wrapping `:start` and `:end` comment lines, but **keep everything in between** — that's your code or content.

After all edits, the file should contain no `joggr` substring. Verify:

```bash
grep -i 'joggr' path/to/doc.md && echo "still references joggr" || echo "clean"
```

### 4. Commit the changes

```bash
git add '*.md' '*.mdx'
git diff --cached --quiet || git commit -m "chore: remove joggr-managed comments"
```

> [!NOTE]
> Skip this step until **after** you've uninstalled the GitHub App. Otherwise Joggr's app may re-inject the comments on its next sync.

## Scripts

Two zero-dependency Node scripts. Require **Node 20+**.

> [!CAUTION]
> `offboard.mjs` rewrites markdown files in place. Each file is copied verbatim to `~/.joggr-docs-offboard-backup/files/<relPath>` before the rewrite. Always run `node offboard.mjs --dry-run` first.

### `offboard.mjs` — strip markers

One pass. Discovers every `.md`/`.mdx` file under the scan root with Joggr markers, prompts once to proceed, then for each file: backup → verify hash → rewrite. Records what it did in a small manifest so `restore.mjs` can undo it.

```bash
node offboard.mjs --dry-run                 # preview, still prompts
node offboard.mjs                           # scan the enclosing git repo
node offboard.mjs --scan ~/Code/your-docs   # scan a specific path
node offboard.mjs --yes                     # auto-confirm the proceed prompt
```

**The script refuses to run** (and tells you to use manual steps) if:

- The scan root is a git repo with uncommitted changes. Commit or stash first.
- A backup directory already exists at `~/.joggr-docs-offboard-backup/`. Either run `restore.mjs` or `rm -rf` it manually.
- A file has unmatched `@joggr:NAME:start` / `:end` markers. Inspect that file by hand.
- A backup's hash doesn't match the original (extremely rare; would indicate disk corruption).
- Any file read or write fails.

The trimmer is:

- **Per-block aware.** Only `warning` and `editLink` blocks have their content removed. Every other block type (`snippet`, `createSnippet`, `autoTarget`, `codeLink`, `createCodeLink`, plus any future types Joggr ships) has its markers removed and content preserved verbatim. This matches `@joggrdocs/bashir`'s upstream behavior and is pinned by unit tests against bashir's fixtures.
- **Code-fence aware.** Markers inside ``` or `~~~` blocks are never touched, so docs that quote a Joggr marker as an example are safe.
- **Stack-balanced.** A `:start` must have a matching `:end` with the same `NAME(PARAMS)`. Unmatched markers cause the script to fail with the file path.

### `restore.mjs` — undo

Reads the manifest and copies each backed-up file back to its original path. Verifies the backup's SHA-256 against the manifest before writing. Refuses to overwrite if the hash doesn't match.

```bash
node restore.mjs --dry-run    # preview
node restore.mjs              # interactive
node restore.mjs --yes        # auto-confirm
```

After all files are restored, you'll be asked whether to delete the backup directory.

The GitHub App install is not touched by `restore.mjs` — reinstall it from <https://github.com/marketplace/joggr> if you want it back.

## Layout

```
offboarding/docs/
├── README.md                    you are here
├── package.json                 npm scripts (offboard, restore, test)
├── install.sh                   bootstrap: download this dir via curl
├── offboard.mjs                 strip Joggr markers (per-file backups)
├── restore.mjs                  undo a previous offboard
└── lib/
    ├── constants.mjs            marker regexes, dirnames
    ├── utils.mjs                trimmer, scanner, manifest IO
    └── __tests__/
        └── utils.test.mjs       trimmer unit tests (node --test)
```

The backup directory lives at `${HOME}/.joggr-docs-offboard-backup/` — peer to `~/.claude/`. Override with `JOGGR_DOCS_OFFBOARD_BACKUP_DIR=/some/path` (used for tests). Layout:

```
${HOME}/.joggr-docs-offboard-backup/
├── manifest.json                what was stripped (for restore)
└── files/<relPath>              verbatim copies of every file we touched
```
